# 久安问道录 Demo 玩法验证清单

## 1. 文档目标

这份清单用于指导当前 demo 的后续玩法验证，重点不是“功能有没有”，而是验证：

- 不同系统是否真的互相作用
- 不同选择是否能导向不同结果
- 玩家是否能理解系统、形成取舍、看到反馈
- 当前实现是否已经足够支撑下一轮扩展

本文默认以当前项目已落地版本为基线：

- 开周选课
- 通识必修 / 专业必修 / 专业选修
- 6 时段日程安排
- 白天活动执行
- 夜间记忆构筑
- 一周总结与评级

## 2. 本轮验证总原则

每次验证尽量遵循这 4 条原则：

1. 固定随机种子或尽量减少变量。
2. 一次只验证一个核心问题。
3. 每组实验至少保留一份结果记录。
4. 优先验证“是否有差异”，再验证“数值是否平衡”。

## 3. 优先验证顺序

建议按以下顺序推进：

1. 选课是否真的改变一周打法
2. 自由时段是否形成真实取舍
3. 夜间记忆系统是否反向影响白天决策
4. 课程本体、作业、随机事件三者是否区分明确
5. 一周节奏是否清楚，玩家是否知道自己在追什么

## 4. 核心验证项

### 4.1 选课差异验证

#### 验证目标

确认不同选课组合会带来不同的一周成长路线，而不是只改了课表外观。

#### 推荐验证组

- 方案 A：偏数术 / 阵法
- 方案 B：偏道法 / 灵物
- 方案 C：偏符法 / 炼器

#### 操作步骤

1. 固定开局原型与随机条件。
2. 分别完成 3 套不同的开周选课。
3. 每套方案按相同风格安排 7 天自由时段。
4. 记录周结算结果。

#### 重点观察项

- 最强技能是否不同
- 悟道点是否拉开差距
- 灵力值是否拉开差距
- 夜间灵块类型分布是否不同
- 最终建筑分布是否不同
- 白天推荐活动是否随课程方向变化

#### 通过标准

- 3 套方案在至少 3 个维度上出现明显差异
- 玩家能在中途感受到“这周的主修方向是什么”

#### 失败信号

- 不同选课下结果高度趋同
- 玩家只需要无脑排同一种自由活动即可
- 夜间记忆系统与选课几乎无关

### 4.2 自由时段取舍验证

#### 验证目标

确认白天自由活动之间存在策略博弈，而不是某一项明显碾压。

#### 推荐验证组

- 全学习流：优先做课业、补学习相关行动
- 全恢复流：优先洗漱、吃饭、恢复类行为
- 全赚钱流：优先打工、节约资源
- 均衡流：学习、恢复、社交、赚钱混搭

#### 操作步骤

1. 固定相同选课方案。
2. 用 4 套不同的自由时段模板跑一周。
3. 记录所有周结算数据。

#### 重点观察项

- 哪类路线的灵力值最高
- 哪类路线的悟道点最高
- 哪类路线最容易疲惫失控
- 哪类路线更容易形成夜间高质量建筑
- 哪类路线在剧情 / 关系推进上更占优

#### 通过标准

- 没有单一路线在资源、成长、记忆、剧情上全部碾压
- 不同路线都能形成“有得有失”的结果

#### 失败信号

- 课业收益压倒性过高
- 恢复类行为只有补数值，没有战略意义
- 社交 / 打工 / 城市探索完全不值得排

### 4.3 夜间记忆系统联动验证

#### 验证目标

确认夜间记忆系统不是附属小游戏，而是能反向塑造白天行为的核心玩法。

#### 操作步骤

1. 固定一套选课方案。
2. 连续跑 5 到 7 天，优先追某一种夜间建筑路线。
3. 观察白天是否会因此被迫调整。

#### 建议验证路线

- 术式楼路线
- 悟理阁路线
- 养神台路线
- 衔接塔优先路线

#### 重点观察项

- 白天课程是否稳定生成对应灵块
- 玩家能否通过形状识别灵块类型
- 玩家能否通过颜色判断落区
- 建成建筑后，第二天是否会影响日程选择
- 为了追某个建筑，玩家是否会主动改变白天安排

#### 通过标准

- 至少存在 1 到 2 条可持续追的建筑路线
- 玩家会为了夜间构筑主动调整白天活动

#### 失败信号

- 夜间建筑只带来弱反馈
- 白天活动与夜间构筑完全脱节
- 灵块辨识仍然需要强依赖文字说明

### 4.4 课程本体与作业差异验证

#### 验证目标

确认“上课”和“做课业”是两个不同层次的玩法，不是同一种收益的重复包装。

#### 操作步骤

1. 跑一组“只上课不补课业”的方案。
2. 跑一组“上课后持续补课业”的方案。
3. 对比 3 天和 7 天结果。

#### 重点观察项

- 作业是否能放大当日主修方向
- 不做作业时，成长是否会更松散
- 做课业是否会明显牺牲其他收益
- 作业是否真的起到“沉淀课程”的作用

#### 通过标准

- 课程更偏输入，作业更偏沉淀与放大
- 玩家能感受到“今天学了什么，晚上要不要补一手”

#### 失败信号

- 作业与课程收益几乎没有结构差别
- 做课业只是在堆数值，没有方向性

### 4.5 随机事件价值验证

#### 验证目标

确认随机事件的作用是打破固定最优解，而不是只增加噪音。

#### 当前建议最少补齐的事件类型

- 正向事件：额外收益或特殊机会
- 负向事件：打断排程、降低收益、改变当日选择
- 替换类事件：挤占原时段

#### 重点观察项

- 随机事件是否能改变当天决策
- 事件对不同 build 的影响是否不同
- 事件是否能和课程 / 作业 / 关系系统联动

#### 通过标准

- 玩家会因随机事件改变原计划
- 事件不只是提示文本，而是确实影响结算或时段安排

#### 失败信号

- 事件只写日志，不改变行为
- 事件影响过弱，玩家完全无视
- 事件影响过强，直接毁掉一整周节奏

### 4.6 一周节奏清晰度验证

#### 验证目标

确认玩家在一周流程中始终知道：

- 现在在做什么
- 为什么这么做
- 做完会得到什么

#### 建议观察时间点

- 开周选课结束后
- 第 1 天白天结束后
- 第 3 天夜间构筑后
- 周结算时

#### 要记录的问题

- 玩家是否知道固定课和自由时段的区别
- 玩家是否知道白天活动如何影响夜间灵块
- 玩家是否知道夜间建筑有什么意义
- 玩家是否能理解周结算为什么是这个结果

#### 通过标准

- 玩家几乎不会问“下一步该干嘛”
- 玩家能说出自己这周在追的方向
- 周结算能解释前面的选择结果

#### 失败信号

- 玩家中途失去目标感
- 玩家把夜间系统当作独立小游戏
- 玩家看不懂周结算与过程的关系

## 5. 推荐实验矩阵

### 5.1 第一轮最值得马上做的实验

建议先做 3 组一周实验：

| 组别 | 选课方向 | 自由时段风格 | 主要验证问题 |
| --- | --- | --- | --- |
| A | 数术 / 阵法 | 偏学习 | 课程方向是否影响成长与灵块 |
| B | 道法 / 灵物 | 偏恢复 / 稳定 | 恢复类路线是否有独立价值 |
| C | 符法 / 炼器 | 偏工坊 / 赚钱 | 实作路线是否形成不同 build |

### 5.2 第二轮实验

在第一轮有明确差异后，再做：

- 同一套选课下对比 4 种日程模板
- 同一套日程下对比 2 种夜间建筑路线
- 引入随机事件后的路线稳定性测试

## 6. 每轮验证必须记录的数据

每次跑完整周，至少记录：

- 开局原型
- 选课结果
- 每日自由时段模板
- 周结算灵力值
- 周结算悟道点
- 最强技能
- 主要夜间建筑类型
- 剧情 / 关系推进情况
- 明显感受到的强项
- 明显感受到的短板

## 8. 当前阶段的通过线

在当前 demo 阶段，只要满足以下几点，就说明玩法验证已经进入下一阶段：

- 不同选课方案能明显导向不同的一周结果
- 不同自由活动路线不存在单一绝对最优解
- 夜间记忆系统会反向影响白天决策
- 课程、作业、随机事件三者职责清楚
- 玩家能理解一周流程并说出自己的 build 方向

## 9. 当前阶段的高风险点

接下来最需要重点防止的问题有：

- 课程收益和作业收益重叠过高
- 课业成为唯一强选项
- 夜间记忆建筑反馈不足
- 选课只影响文案，不影响真实 build
- 随机事件只是日志，没有策略意义
- 周结算数字有变化，但玩家感知不到“我为什么赢 / 为什么亏”

## 10. 一句话建议

下一步不是优先继续加内容，而是先用这份清单验证：**现在已有的系统是否已经能稳定地产生不同打法、不同节奏和不同结果。** 如果差异还不够明显，就应该先拉开差异，再继续扩玩法。
